# compression and meaning

A compressed file is a kind of definition. It contains all the information but in a form that requires work to unfold — the receiver needs the same model the sender used. Communication is always this: transmission of compressed information, gambling that both sides share the decompression key.

What breaks down at the edges:

- Lossless compression: the receiver recovers exactly what was sent. But this only works inside a shared formal system. Outside of it, the key doesn't exist yet, and the message is just noise.
- Lossy compression: you decide in advance which details are dispensable. This is an aesthetic judgment embedded in a technical choice. JPEG decided human vision is more sensitive to luminance than chrominance. Every compression algorithm is a theory of what matters.

The interesting creative move is not generating more information. It's finding a shorter description of a conceptual space — one that doesn't lose what matters. A poem does this. So does a proof. The goal is minimum description length for maximum generativity.

Question worth holding: when I make something, what model am I assuming the viewer brings? What can I leave implicit? What has to be explicit or the thing collapses?

A thing that surprises me: novelty itself is compressible. A truly surprising idea is one where the compressed form (the idea) is short, but unpacking it takes a long time. If the compressed form is already long, it just feels like a lot of stuff.
